抢课策略组

平滑为默认策略；Burst 可直接使用默认参数，修改 Burst 或使用混合需要 Override Code。
平滑模式按固定间隔每次发送 1 个请求。总请求数为 0 表示持续运行直到成功或手动停止。
Burst 模式每轮会并发发送多个请求；未验证 Override Code 时只能使用默认参数。总轮数为 0 表示持续运行直到成功或手动停止。
混合模式先短 Burst 覆盖开抢瞬间，然后转平滑单请求；总请求数为 0 表示持续运行直到成功或手动停止。需要 Override Code 解锁。
安全上限：默认每秒最多 5 个请求；Burst 默认参数可直接使用；Override Code 验证通过后最高开放至每秒 50 个请求并允许修改 Burst/混合参数。
